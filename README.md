# onちゃんの間違い探しゲーム（HTB秋の大感謝祭展示）

## 概要
このゲームは、似ている絵の違いを探して見つけた間違いの数を競う間違い探しゲームです。  
一部の問題には、絵の一部分が出たり消えたりする仕掛けがあり、最後にはランキングに反映されます。  
HTB秋の大感謝祭では、３日間で約1000人の来場者に遊んでもらいました。

## 遊び方
1. 画面に表示される絵を見て、違う箇所をクリック。  
2. 一部の問題では、絵の一部が消えたり出たりする仕掛けもあり、見逃しやすい箇所に注意。  
3. 問題が難しい場合はスキップも可能。  
4. 違いを見つけた数が記録され、最終的にはランキングに反映。

### プログラム面
- **タイトル画面でのプレイヤー名管理**  
  - TMP_InputField で名前入力を受け付け  
  - 未入力時は「名無し1,2…」の自動生成  
  - 入力文字数を6文字に制限し、PlayerPrefs に保存  

- **オフラインランキングの実装**  
  - ゲーム終了後にスコアを保存し、過去スコアと比較して高い方を保持  
  - 上位10件を表示、プレイヤー自身の順位も表示  
  - PlayerPrefs を使った軽量なデータ管理でサーバ不要  

- **ゲーム体験と連動した工夫**  
  - スコア更新がリアルタイムでUIに反映される  
  - ランキングにより、達成感と競争要素を強化  

### デザイン・演出面
- 家族連れやお子様がターゲットのイベントでの展示だったため、なるべく漢字を使わず文章を構成しました  
- onちゃんのブランドイメージを損なわないよう、セリフは公式に忠実に再現しました  
- 全体的なデザインは、お子様の目を引くかわいらしいデザインにしました

### 課題と改善案
- **課題1：来場者の様子に応じて難易度を柔軟に調整できなかった**
  - 改善案：時間制限などのパラメータをリアルタイムで変更できる仕組みを導入することで、幅広いプレイヤー層に合わせて難易度を調整できるようにする。
  
- **課題2：来場者の回転率が悪かった**
  - 改善案：ランキング登録時の名前入力を「10位以内のプレイヤーのみに限定」することで、プレイ後の待ち時間を短縮し、回転率を向上させる。

- **課題3：一部分が変わっていく問題の正答率が極端に低かった**
  - 改善案：実際の番組仕様をそのままゲームに適用したことで難易度が過度に高くなっていました。ゲームとして適切な形に落とし込めるよう再設計する必要があった。ただし、イベント開催中に即座に　仕様変更して改善できた点は良かった。
